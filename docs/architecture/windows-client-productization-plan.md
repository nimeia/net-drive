# Windows 客户端产品化计划

## 目标

把当前以测试为主的 Windows 客户端，演进为可安装、可配置、可挂载、可诊断、可恢复的产品化客户端。

## Epic 概览

| Epic | 目标 | 优先级 | 主要依赖 | 验收标准 |
| --- | --- | --- | --- | --- |
| E1 | 真实挂载运行时与 WinFsp 生命周期 | P0 | Iter 15 callback host 基线 | 能真实 mount / unmount，状态机稳定，Explorer 可见 |
| E2 | Windows 产品化主界面 | P0 | E1 运行时状态、E3 配置 | 具备 Dashboard / Mounts / Settings / Diagnostics 四块主流程 |
| E3 | 配置持久化与凭据安全 | P0/P1 | 无 | 支持多 Profile、配置恢复、敏感信息安全存储 |
| E4 | 托盘、后台驻留与通知 | P0/P1 | E2 UI 主壳、E7 状态事件 | 可后台驻留、托盘控制挂载、展示关键通知 |
| E5 | 诊断、自检与日志 | P0/P1 | E1/E7 运行时事件 | 有日志、自检、错误映射和诊断导出 |
| E6 | 安装、升级、卸载与发布 | P1 | E5 依赖检查、构建脚本 | 正式安装包可安装 / 升级 / 卸载 |
| E7 | 连接、会话恢复与自动重连 | P1 | clientcore 状态与恢复基线 | 断网恢复、认证恢复、启动恢复上次挂载 |
| E8 | 质量保障与测试体系 | P0/P1 | 各 Epic 核心逻辑 | 核心状态机、配置层、安装链路有自动化与 smoke 验证 |
| E9 | 文件系统语义与性能优化 | P1/P2 | E1 真实挂载 | Explorer / 编辑器兼容性和性能有持续收口 |
| E10 | 增强功能与企业化能力 | P2/P3 | E3/E5/E6 | 模板配置、静默安装、策略约束等能力逐步补齐 |

## 推荐迭代顺序

### 阶段 A：先把主链跑通
1. E1 真实挂载运行时
2. E2 主界面骨架
3. E3 配置持久化与 Profile 管理
4. E4 托盘基础能力
5. E5 日志与自检首页
6. E8 配置层 / 状态机单测

### 阶段 B：把产品做稳
1. E7 自动重连 / 启动恢复
2. E5 诊断导出 / 错误码映射
3. E6 安装升级卸载
4. E8 Windows 主机 smoke / 发布验收

### 阶段 C：增强与企业化
1. E2 首次使用向导 / Mounts 详情页
2. E9 Windows 文件语义与性能收口
3. E10 企业化配置模板 / 策略 / 静默部署

## Iter 18 本轮选择

本轮先做 E3 的第一段：

- 增加 Win32 客户端配置持久化基线
- 支持 Profile 保存 / 加载 / 删除
- 启动时恢复最近一次激活 Profile
- 为配置存储层补跨平台单测

这样可以先把“测试窗体”推进到“可持续使用的客户端控制台”，再继续往真实挂载和主界面骨架演进。
